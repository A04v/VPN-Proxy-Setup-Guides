# MTProxy + Fake TLS на VPS (Docker, nineseconds/mtg)

## Что это и зачем

MTProto Proxy — штатный механизм Telegram для обхода блокировок. Работает только
для Telegram, не требует установки дополнительных приложений — всё настраивается
прямо в Telegram.

**Fake TLS** маскирует трафик под обычный HTTPS к легитимному сайту. Для DPI
(систем глубокой инспекции пакетов) это выглядит как обычное посещение, например,
`ya.ru`. Обычный MTProxy без Fake TLS имеет характерные паттерны трафика, которые
DPI легко обнаруживает и замедляет — особенно медиа.

Используем образ **[nineseconds/mtg](https://github.com/9seconds/mtg)** —
Go-реализация, активно поддерживается, образ весит менее 3.5 МБ, встроенная
защита от replay-атак и статистическая имитация TLS-профиля реального сайта.

---

## Почему важна локация VPS

Это контринтуитивный, но принципиальный момент.

**Зарубежный VPS (Швеция, Германия и т.д.) — плохой выбор** для пользователей
внутри России:

```
Ты (Россия) → [Граница РФ + ТСПУ] → VPS (за рубежом) → Telegram
                      ↑
              Здесь жёсткая фильтрация.
              Трафик до прокси режется ещё до того,
              как он успевает замаскироваться.
```

**Российский VPS — правильный выбор:**

```
Ты (Россия) → VPS (Россия) → [Граница РФ] → Telegram DC (Амстердам)
                   ↑                ↑
        Минимальная фильтрация   Прокси уже замаскировал трафик
        внутри страны            под HTTPS до выхода за рубеж
```

DPI и ТСПУ работают преимущественно на точках обмена трафиком на границе сети.
Трафик между российскими серверами фильтруется значительно слабее.

### Как проверить ноду перед установкой

**Это обязательный шаг.** Не все хостеры пускают трафик до DC Telegram даже
с российских серверов. Проверяй сразу после получения доступа к новой ноде:

```bash
curl -v https://149.154.167.51:443 --max-time 5
```

**Нода подходит**, если в выводе есть строки вида:
```
* Connected to 149.154.167.51 ...
* TLSv1.3 (OUT), TLS handshake, Client hello
* error:0A00010B:SSL routines::wrong version number
```
Ошибка `wrong version number` — это норма. Telegram DC отвечает не на стандартный
HTTPS, а на свой протокол. Главное — соединение устанавливается.

**Нода НЕ подходит**, если:
```
* OpenSSL SSL_connect: SSL_ERROR_SYSCALL
```
Это активный сброс соединения — хостер режет трафик до Telegram.

## Требования

- VPS с Ubuntu 22.04 / 24.04, **российская локация**
- Docker (установлен)
- Свободный порт **443** (для корректной работы Fake TLS нужен именно 443)

---

## Шаг 1. Установка Docker

```bash
curl -fsSL https://get.docker.com | sh
```

Проверить:

```bash
docker --version
```

---

## Шаг 2. Проверь что порт 443 свободен

```bash
ss -tlnp | grep 443
```

Если вывод пустой — порт свободен. Если занят — найди и останови процесс:

```bash
systemctl stop nginx   # или apache2, caddy — смотри по ситуации
```

---

## Шаг 3. Сгенерируй секрет с Fake TLS

```bash
docker run --rm nineseconds/mtg:2 generate-secret --hex ya.ru
```

Получишь строку вида:

```
ee1c7666e0b011ee3989f0d4078844102d79612e7275
```

Префикс `ee` означает что секрет содержит Fake TLS с закодированным доменом
маскировки. Сохрани эту строку.

### Выбор домена для маскировки

Домен должен быть популярным и точно не заблокированным в РФ:

| Домен | Почему подходит |
|---|---|
| `ya.ru` | Яндекс — явно в белом списке |
| `vk.com` | ВКонтакте — аналогично |
| `gosuslugi.ru` | Госсервисы — не заблокируют никогда |
| `sberbank.ru` | Сбербанк |
| `1c.ru` | Распространён у бизнеса |

> ⚠️ **Не используй** `google.com`, `cloudflare.com` — они частично ограничены
> в РФ, и провайдеры легко видят что твой VPS не принадлежит Google.
> Выбирай домен, логически совместимый с локацией твоего сервера.

---

## Шаг 4. Запусти контейнер

```bash
docker run -d --name mtproto-proxy --restart unless-stopped -p 443:443 nineseconds/mtg:2 simple-run -n 1.1.1.1 -i prefer-ipv4 0.0.0.0:443 ВАШ_СЕКРЕТ
```

**Пример:**

```bash
docker run -d --name mtproto-proxy --restart unless-stopped -p 443:443 nineseconds/mtg:2 simple-run -n 1.1.1.1 -i prefer-ipv4 0.0.0.0:443 ee1c7666e0b011ee3989f0d4078844102d79612e7275
```

**Пояснение флагов:**

| Флаг | Значение |
|---|---|
| `-p 443:443` | Слушаем на порту 443 — обязательно для Fake TLS |
| `simple-run` | Режим прямого подключения к Telegram |
| `-n 1.1.1.1` | DNS-резолвер (Cloudflare) |
| `-i prefer-ipv4` | Предпочитать IPv4 |
| `0.0.0.0:443` | Слушать на всех интерфейсах |

---

## Шаг 5. Проверь что запустилось

```bash
docker ps
```

Контейнер `mtproto-proxy` должен быть в статусе `Up`.

```bash
docker logs mtproto-proxy
```

Пустой вывод — норма для `mtg`. Если есть ошибки — скинь их.

Узнай IP сервера:

```bash
curl -s ifconfig.me
```

---

## Шаг 6. Подключись в Telegram

Твоя ссылка (замени IP на свой):

```
tg://proxy?server=ВАШ_IP&port=443&secret=ВАШ_СЕКРЕТ
```

Отправь её себе в Saved Messages и открой на телефоне — Telegram предложит
подключиться автоматически.

Либо вручную: **Настройки → Данные и хранилище → Прокси → Добавить прокси**

- Тип: **MTProto Proxy**
- Server: IP сервера
- Port: `443`
- Secret: секрет с префиксом `ee...`

## Мониторинг с Uptime Kuma

Uptime Kuma — self-hosted дашборд для мониторинга с уведомлениями в Telegram.
Разворачивается в отдельном Docker-контейнере на том же сервере, не конфликтует
с MTProxy.

### Установка

```bash
docker run -d --name uptime-kuma --restart unless-stopped -p 3001:3001 -v uptime-kuma:/app/data louislam/uptime-kuma:1
```

Открой в браузере:

```
http://ВАШ_IP:3001
```

При первом входе создай логин и пароль.

### Настройка уведомлений в Telegram

1. Напиши [@BotFather](https://t.me/BotFather) команду `/newbot` — получишь токен вида `123456789:AAF...`
2. Узнай свой Chat ID через [@userinfobot](https://t.me/userinfobot)
3. В Uptime Kuma: **Settings → Notifications → Add Notification**
   - Notification Type: `Telegram`
   - Bot Token: токен от BotFather
   - Chat ID: твой ID
   - Нажми **Test** — должно прийти тестовое сообщение
   - **Save**

### Добавление монитора MTProxy

**Add New Monitor:**

- Monitor Type: `TCP Port`
- Friendly Name: `MTProxy Fake TLS`
- Hostname: IP твоего сервера
- Port: `443`
- Heartbeat Interval: `60` (проверка каждую минуту)
- Подключи созданное уведомление
- **Save**

Uptime Kuma будет проверять порт 443 каждую минуту и пришлёт сообщение в Telegram
если прокси перестанет отвечать.

### Полезные команды

```bash
# Статус
docker ps

# Логи
docker logs uptime-kuma

# Перезапуск
docker restart uptime-kuma
```

## Что делать если тормозит или не подключается

1. **Сначала проверь доступность DC Telegram** с сервера (команда из раздела
   «Как проверить ноду») — если `SSL_ERROR_SYSCALL`, хостер режет трафик,
   нужно менять ноду или хостера.

2. **Смени домен маскировки** — пересгенерируй секрет с другим доменом
   (`gosuslugi.ru`, `1c.ru`) и пересоздай контейнер.

3. **Звонки через MTProxy не работают** — это архитектурное ограничение
   протокола. Для звонков нужен VPN.
